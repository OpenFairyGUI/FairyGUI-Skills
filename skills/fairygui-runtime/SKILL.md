# FairyGUI Runtime Skill

## Workflow

1. Confirm target engine and language (Unity C#, UE C++, Cocos, or custom runtime).
2. Identify the runtime task: package loading, component binding, events, data flow, or animation control.
3. If the task needs deeper details, read the relevant reference file:
   - Runtime setup: references/runtime-setup.md
   - Component binding: references/component-binding.md
   - Events and animation: references/events-animation.md
   - Package loading: references/loading-packages.md
4. Provide code-level steps and highlight required API calls and lifecycle hooks.

## Output expectations

- Provide minimal, working code snippets.
- Note engine lifecycle constraints (e.g., init order, async load).
- Ask for the runtime version and any custom wrappers.
